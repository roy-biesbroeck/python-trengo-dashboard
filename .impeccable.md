## Design Context

### Users
Support team members viewing a **wall-mounted, always-on display** in a team room. The dashboard shows live Trengo ticket statistics — users glance at it throughout the day to stay aware of queue health, team workload, and ticket aging. Readability at a distance and at a glance is critical. The UI language is Dutch (nl-NL).

### Brand Personality
**Friendly, Warm, Approachable** — the dashboard should feel inviting and human, not cold or intimidating. It serves a support team, so it should reflect the same approachability they bring to customers.

### Aesthetic Direction
- **References:** Notion and Stripe — clean, friendly, well-structured with clear visual hierarchy
- **Anti-references:** Avoid dense/dark monitoring tools (Grafana/Datadog style) or cold corporate dashboards
- **Theme:** Light mode with warm neutrals
- **Typography:** System font stack (current), with generous sizing for wall-display legibility
- **Layout:** Spacious cards, clear groupings, breathing room between sections

### Color System
The existing palette is well-established and should be preserved:
- **Primary:** #3b82f6 (blue) — actions, totals, brand accent
- **Status semantic:** green (#16a34a) = new/unassigned, amber (#d97706) = assigned, red (#dc2626) = critical
- **Closed tickets:** #8b5cf6 (purple)
- **Age gradient:** blue → green → lime → amber → orange → red (fresh → stale)
- **Neutrals:** #f1f5f9 background, #ffffff cards, #1e293b text, #e2e8f0 borders

### Accessibility Requirements
- **WCAG AA compliance** — all text must meet minimum contrast ratios (4.5:1 for normal text, 3:1 for large text)
- Ensure color is never the only indicator of status (pair with labels/icons)
- Keyboard navigation support
- Respect `prefers-reduced-motion` for animations
- Consider color blindness — the age gradient and status colors should remain distinguishable

### Design Principles

1. **Glanceable at 3 meters** — Wall-mounted context means large type, strong hierarchy, and high contrast. If you can't read it from across the room, it's too small.

2. **Warm, not sterile** — Use friendly language, generous spacing, and subtle warmth in the palette. This is a team tool, not a server monitoring console.

3. **Color tells the story** — Status colors are the primary communication channel. Preserve their semantic meaning and ensure they work for colorblind users.

4. **Less is more** — Every element must earn its space. Follow Notion/Stripe: clean structure, clear hierarchy, no decoration for decoration's sake.

5. **Accessible by default** — WCAG AA is the floor, not the ceiling. Contrast, keyboard nav, motion preferences, and screen reader support are not afterthoughts.

### Tech Stack
- **Backend:** Python/Flask
- **Frontend:** Vanilla HTML/CSS/JS (no framework)
- **Charts:** Chart.js 4.4.0
- **Styling:** CSS custom properties (no preprocessor, no utility framework)
- **No build step** — all changes are direct to HTML/CSS/JS files
